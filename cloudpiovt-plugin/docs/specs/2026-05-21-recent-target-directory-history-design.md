# 最近目标路径设计

## 背景

当前扩展已经支持两类目标目录来源：

- File System Access 目录句柄
- Native Host 返回的绝对路径

页面级目录快照与全局默认目录已经拆分，确保“已打开页面”不会被后续默认目录更新覆盖。新增最近路径功能时，不能破坏这套快照语义。

## 目标

为弹窗增加“最近路径”快捷入口，降低重复选择相同目录的成本，并为绝对路径区域增加展开/收起交互。

## 范围

### 包含

- 在“绝对路径”区域增加展开/收起能力
- 展开后展示当前绝对路径、复制按钮、最近 5 条路径、重新选择按钮
- 点击历史路径时，直接切换为当前页面目录，并同步更新后续新页面默认目录
- 支持从最近路径中移除单条记录
- 当前正在使用的路径在列表中高亮并标记“当前”

### 不包含

- 不为 File System Access 句柄生成伪绝对路径
- 不新增收藏夹、命名别名或分组管理
- 不改变现有页面快照与默认目录继承规则

## 设计决策

### 1. 历史记录只保存 Native 绝对路径

File System Access 句柄无法稳定给出绝对路径，因此最近路径只记录 Native Host 返回的绝对路径。句柄模式仍按现有行为工作，但不会写入最近路径。

### 2. 历史记录是快捷入口，不是新的主状态源

历史记录只负责提供可点击的候选路径。真正切换目录时，仍然复用现有 `saveNativeTargetDirectorySelection(pageType, directoryPath, pageScope)` 流程，确保：

- 当前页面快照更新
- 新页面默认目录更新
- 旧页面快照不被回写

### 3. 历史记录全局保存，但保留 pageType 元数据

最近路径默认展示最近 5 条，不按页面类型拆分 UI。每条记录保留 `pageType` 与 `lastUsedAt`，为后续按页面类型筛选或排序扩展预留空间。

## 交互说明

### 收起态

- “绝对路径”一行显示当前目录名或“未保存”
- 右侧箭头表示可展开
- 点击整行切换展开/收起

### 展开态

- 第一行显示完整绝对路径，没有路径时显示“未保存”
- 提供“复制路径”按钮
- 展示最近路径列表
- 提供“重新选择”按钮，继续走现有目录选择流程

### 最近路径项

- 主文本只显示目录名
- `title` 展示完整路径
- 点击主区域直接切换
- 右侧小按钮可移除该条历史
- 当前路径显示“当前”标记并高亮

## 错误与边界

- 历史路径去重，按最近使用时间排序
- 历史路径上限固定为 5 条
- 用户移除历史记录时，只删除快捷记录，不清空当前目录绑定
- 点击当前路径对应的历史项时不重复写入状态，只提示当前已在使用
