    推荐流程

    1. 先配编辑器和原生助手

    先填写 VS Code 与 IDEA 的 exe 路径，并确认上面的原生助手状态可用。原生助手负责选择应用、返回目录绝对路径，以及直接拉起本地编辑器。

    2. 再绑定当前页面目录

    回到弹窗点击“更新目标目录”，为当前页面类型建立目录快照。重新选择目录时，会影响当前页面和后续新页面默认值，但不会回头覆盖已经打开的旧页面。

    3. 前端页面用抓取写入和前端回写

    在在线开发页面使用“前端抓取写入”，表单、列表和默认页面类型都会按各自固定文件名写入目标目录；本地修改后，再用“前端回写”同步回页面。

    4. 业务规则页面单独处理

    在业务规则开发页面使用“业务规则抓取写入”和“业务规则回写”。插件会优先按 Monaco model URI 或 Java 类名解析 .java 文件名，并按同名 model 回写。

    5. 用复制和最近路径做排查

    弹窗顶部的 VS Code / IDEA 按钮会直接打开当前页面绑定的目标目录；“复制链接”“复制路径”和最近路径切换适合在排查目录命中、页面来源或多人协作时快速对齐上下文。

    常见排查

    编辑器打不开

    优先检查应用路径是否配置正确、原生助手是否已连接，以及当前页面是否能拿到目录绝对路径。编辑器按钮依赖原生助手返回目标目录路径后再执行本地打开。

    最近路径为空或不更新

    最近路径只记录原生助手返回的绝对路径，不记录 File System Access 目录句柄。若本次目录选择没有拿到绝对路径，就不会出现在最近路径列表中。

    提示没有对应的 .java 文件

    这通常不一定是目录里真缺文件，更常见的原因是当前页面同时打开了多个业务规则，导致本次抓取或回写命中了错误的 Monaco model。先关闭多余业务规则再试一次。
