# 全局 Agent 规则

## 1. 硬性环境约束

- 文件编码固定使用 UTF-8，不得修改为 GBK、ANSI 或其他编码。
- PowerShell 固定使用 PowerShell 7，不得使用 Windows PowerShell 5。
- 执行命令前，优先确认当前终端、路径、编码与项目环境。
- 设计到文件是需要使用到路径的，尽量不使用绝对路径，使用相对路径。


---

## 2. 指令优先级

1. 用户在当前会话中的明确要求
2. 当前项目根目录及相关子目录中的 AGENTS.md
3. 项目已有文档、代码风格、目录结构、命名约定
4. 本全局 AGENTS.md

说明：

- 本文件只定义通用工程纪律、环境约束、默认行为与沟通规则。
- 项目 AGENTS.md 负责项目实现规则。
- 若项目规则与本文件冲突，除硬性环境约束外，优先执行项目规则。

---

## 3. 默认工作方式

- 默认采用满足质量要求的最短路径。
- 小任务默认轻量执行。
- skill 仅在与任务高度相关时启用。
- 不做与任务无关的重构。
- 不修改任务范围外文件。
- 先理解项目规则，再执行任务。

---

## 4. 任务分流

### 只读任务

以下任务可直接处理：

- 代码解释
- 架构分析
- 报错定位
- 命名建议
- 文档分析
- 只读排查

要求：

- 结论必须清晰
- 根因必须可追溯
- 不得虚构验证结果

### 实现任务

涉及以下内容时，应提升验证与工程纪律：

- 新功能
- Bug 修复
- 行为变更
- 重构
- API 修改
- 数据处理逻辑修改

---

## 5. 代码与注释规则

### 通用原则

- 保持项目原有结构、风格与命名。
- 不引入项目不存在的依赖。
- 不硬编码密钥、Token、凭证。
- 数据库访问必须参数化。

### 中文注释要求

生成或修改 JS、CSS、Java 等代码时：

必须补充：

- 方法业务用途
- 关键变量含义
- 核心业务逻辑
- 状态流转原因
- 外部调用原因
- 字段/表单/子表对应中文含义

禁止：

- 空洞翻译式注释
- 无意义逐行注释
- 用注释掩盖混乱代码

---

## 6. 验证规则

- 不得虚构执行结果。
- 未验证不得声称“已完成”“可提交”“可合并”。
- 关键验证无法执行时，必须明确原因。
- 修改代码后需检查：
  - 功能影响范围
  - 回归风险
  - 中文注释
  - 文档同步

---

## 7. 安全规则

- 不运行危险删除命令。
- 不执行 git reset 等破坏性命令，除非用户明确要求。
- 不使用非 Git 工具修改 .git。
- 不拼接不可信 shell 命令或 SQL。
- 不终止非当前任务产生的进程。

---

## 8. 沟通与输出

- 默认使用简体中文。
- 技术术语可使用英文。
- 代码标识符保持英文。
- 执行类任务强调：
  - 当前动作
  - 进度
  - 验证结果
- 分析类任务强调：
  - 结论
  - 根因
  - 依据
  - 风险
- 输出时要输出当前回答使用到那些规则或者skills
