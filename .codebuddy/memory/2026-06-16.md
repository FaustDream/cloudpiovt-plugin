## 诊断分析：用户报告的子表编码缺失根因（2026-06-16 14:20）

- **诊断文件**：`cloudpiovt-plugin-diagnostic-2026-06-16T06-19-31-788Z.json` + `cloudpiovt-plugin-log-2026-06-16T06-19-24-429Z.txt`
- **页面**：修试公司员工变动会签审批流程（氚云表单设计器）
- **症状**：子表控件编码缺失 5 个
- **根因定位**：
  1. **DOM层**：子表 `.sheet-control` 元素不渲染 `data-code`/`data-control-code` 属性（13个控件均为空）
  2. **Vue状态层**：部分控件的 `control.code` 在 `collectObjects(depth=5, keys=80)` 范围内不可达
  3. **Catalog回填层（核心缺陷）**：`buildSheetFieldCodeCatalog` 对第一个子表（8个控件）只捕获到3个编码条目（F0000107、F0000108、FormGridView），`childrenOf` 按 index 回填但不验证名称匹配，导致 index 0-2 被错误赋值了不对应的编码，index 5-7 因越界编码为空
  4. **设计缺陷**：回填不验证 `displayName` 匹配，单纯依赖 index 顺序
- **第二个子表（人员退休信息）**：5个控件全部成功，因为 catalog 条目充足且名称匹配
- **修复方向**：增强 catalog 扫描范围、childrenOf 增加名称匹配验证、增加 Vue 深查 depth 或 JSON.stringify 扫描

## 修复实现：通过 allControls 全局状态直接读取子表自定义编码（2026-06-16 16:27）

- **方案确定**：经过 grill-me 深度盘问，确定使用 Vue `allControls` 全局对象作为主要数据源，现有 DOM 扫描作为兜底
- **关卡发现**：默认编码（F0000107）能通过 DOM/innerHTML 扫描到，但自定义编码（changeUser、changeDate）只在 allControls 的 Vue 全局状态中
- **修改文件**：
  - `popup.js`：
    - 新增 `findDesignerAllControls()`：从 `.designer.web.__vue__.allControls` 或沿子表控件祖先链读取
    - 新增 `extractControlsFromAllControls(rawAllControls, params)`：按 `sheetCode.fieldCode` 点号格式分类构建 controls 数组（容器 vs 子字段、孤儿子字段兜底）
    - 修改主 `try` 块：优先尝试 allControls，成功则直接返回（`source: "allControls"`）；失败则回退到原有 DOM 扫描 + catalog 回填逻辑
  - `lib/h3yun-code.js`：
    - `stripH3yunSheetCodePrefix`：正则从 `/^[A-Za-z0-9]+\.(F[A-Za-z0-9]+)$/` 扩展为 `/^[A-Za-z0-9]+\.([A-Za-z0-9_]+)$/`，支持自定义编码（changeUser、changeDate 等）
    - `buildH3yunFromCodeContent`：新增 `数据来源` 行（allControls/dom）；新增主表控件和子表控件的附加属性：默认值、关联表单（boschemaCode）、显示规则、选项值（DefaultItems）
    - 新增 `appendOptionalField()` 和 `appendOptionalItemsField()` 辅助函数
  - `tests/h3yun-code.test.js`：更新 strip 测试断言，自定义编码也应剥离 D 前缀
- **测试结果**：3 files / 146 tests 全部通过
- **关键数据映射**（allControls → controls 格式）：
  - `DisplayName` → displayName
  - `type`/`ControlKey` → controlKey
  - `key（点号后部分）` → code（子表字段编码，自动剥离 D 前缀）
  - `DefaultValue` → defaultValue
  - `BOSchemaCode` → boschemaCode
  - `DefaultItems` → defaultItems（选项数组）

## Git SSL 证书问题修复（2026-06-16 16:33）

- **问题**：`git pull --tags origin master` 报错 `error setting certificate file: C:/Program Files/Git/mingw64/etc/ssl/certs/ca-bundle.crt`
- **根因**：Git 2.45.1 系统级 `http.sslCAInfo` 指向的证书文件不存在（卸载残留路径）
- **修复**：`git config --global http.sslBackend schannel`，将 SSL 后端从 openssl 切换到 Windows 原生 schannel
