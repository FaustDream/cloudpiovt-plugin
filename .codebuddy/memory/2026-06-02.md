# 2026-06-02 工作日志

## 修复 cloudpiovt-plugin Native Host 安装脚本编码问题

- **问题**：`install-native-host.cmd` 运行报错，`cmd.exe` 将中文标点 `，` `。` 的 UTF-8 字节序列解析为多个命令碎片。
- **根因**：`.cmd` 文件含非 ASCII 字符（中文标点），`cmd.exe` 对 UTF-8 without BOM 的解析存在兼容性问题。
- **修复**：移除所有非 ASCII 字符，改为英文提示。文件现为纯 ASCII（UTF-8 without BOM）。
- **文件**：`cloudpiovt-plugin/scripts/install-native-host.cmd`
- **建议**：如需中文提示，需存为 UTF-8 with BOM 并确保 `chcp 65001` 生效，但兼容性较差，建议保持英文提示。

## .NET → Rust Native Host 重写（方案 A）

- **动机**：.NET 自包含发布包 ~200MB，即使改为依赖系统 .NET Runtime 仍需用户安装 Runtime。
- **方案**：用 Rust 重写 Native Host，静态编译为单个 .exe，无任何运行时依赖。
- **项目位置**：`cloudpiovt-plugin/native-host-rust/`

### 实现结果
- **15 个测试全部通过**
- Release 二进制大小：**391 KB**（vs .NET 的 200MB，缩小约 500 倍）
- 无需 .NET Runtime，用户零依赖安装
- 功能完全等价：ping、文件读写（含子目录创建）、编辑器启动、文件/文件夹选择对话框

### 技术细节
- 使用 TDD 开发：先写测试，再实现
- `native-dialog` crate 实现 Windows 原生文件/文件夹对话框
- 路径安全：防止目录遍历攻击（`..` 检测）
- 所有 PowerShell 脚本中文注释改为英文（避免 PowerShell 5 兼容性问题）
