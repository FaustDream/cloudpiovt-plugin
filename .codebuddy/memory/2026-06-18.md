# 2026-06-18

## 氚云 allControls 获取、提取 & FromCode 输出完善 (1.6.1)

### allControls 搜索
- **根因**：allControls 不在 `.designer.web.__vue__` 的直接属性上
- **修复**：`findDesignerAllControls` 改为递归搜索：`__vue__` 直接属性 → `$data` → `$options` → 递归 `$children` → 子表祖先链 → 全 DOM 元素遍历
- 现在数据来源显示 "allControls" 而非 "dom"

### 数据读取
- **根因**：氚云实际数据结构 `entry.options.DisplayName/DefaultValue/BOSchemaCode/DisplayRule/DefaultItems`，此前代码直接读 entry 顶层
- **修复**：新增 `readAllControlField(entry, fieldName)`，优先从 `entry.options[fieldName]` 读取，不存在时回退到 `entry[fieldName]`
- 主表/子表/孤儿子表控件均统一使用此函数

### 附加修复
- **控件类型**：`ControlKey`（options 内字符串如 FormTextBox）优先于 `type`（顶层数字如 301）
- **显示规则**：新增 `safeText()` 将对象转为 JSON 字符串，替代 `text()` 解决 `[object Object]` 问题
- **选项值**：主表控件和孤儿子表控件增加 `defaultItems` 提取和 FromCode 输出

### 调试过程
- 调试代码已全部清理（`_debugAllControls`、`_debugRawEntry`、`buildDebugRawEntryLines`）
- 版本号保持 1.6.1，此为同一版本的补充修复
